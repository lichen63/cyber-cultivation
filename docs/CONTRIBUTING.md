## How to Contribute

1.  **Fork the Repository**: Create a fork of the repository on GitHub.
2.  **Create a Branch**: Create a new branch for your feature or bug fix.
    ```bash
    git checkout -b feature/my-new-feature
    ```
3.  **Make Changes**: Implement your changes. Ensure your code follows the existing style and conventions.
4.  **Test**: Run the project and ensure everything works as expected.
5.  **Commit**: Commit your changes with a clear and descriptive commit message.
    ```bash
    git commit -m "Add some amazing feature"
    ```
6.  **Push**: Push your branch to your fork.
    ```bash
    git push origin feature/my-new-feature
    ```
7.  **Pull Request**: Submit a Pull Request to the main repository.

## Code Style

- Please follow the standard Flutter/Dart coding conventions.
- Ensure your code is formatted using `dart format`.

## Reporting Issues

If you find a bug or have a feature request, please open an issue in the issue tracker.
